What is DOM and what is the purpose of DOM?

The Document Object Model (DOM) is a programming interface for HTML documents. It represents the structure of a document as a tree-like model, where each element in the document (such as an HTML tag, attribute, or text node) is represented as an object in the tree. The DOM allows developers to manipulate and interact with HTML documents and its purpose is to create dynamic and interactive web pages.


How will you select HTML elements using DOM? Name the DOM methods.

To select HTML elements using DOM use: document.getElementsByName()

DOM methods:

  • getElementById(): Selects an element with a specific ID attribute.
  • getElementsByClassName(): Selects elements with a specific class name.
  • getElementsByTagName(): Selects elements with a specific tag name.
  • querySelector(): Selects first element that matches a specific CSS selector.
  • querySelectorAll(): Selects all elements that match a specific CSS selector.
  • createElement(): Creates a new element with the specified tag name.
  • removeChild(): Removes a child node from an element.
  • replaceChild(): Replaces a child node with a new node.
  • innerHTML(): Gets or sets the HTML content of an element.
  • style(): Gets or sets the CSS style of an element.
  • addEventListener(): Attaches an event listener to an element.
  • removeEventListener(): Removes an event listener from an element.
  • getAttribute(): Gets the value of a specific attribute of an element.
  • setAttribute(): Sets the value of a specific attribute of an element.

What is event bubble?

Event bubbling is a type of DOM event propagation where the event first triggers on the innermost target element, and then successively triggers on the ancestors of the target element in the same nesting hierarchy till it reaches the outermost DOM element or document object.


What is a callback function and why will you use it?

A callback function is a function that is passed as an argument to another function and is intended to be called after the first function has completed its task. The primary purpose of a callback function is to allow the calling function to execute some logic after the called function has completed its work. Callbacks are commonly used in asynchronous programming, event handling, and functional programming to provide a way for a function to delegate the execution of certain logic to another function. They are often used in situations where a function needs to perform some asynchronous operation, such as fetching data from a server or waiting for user input, and then execute some logic when the operation is complete.
